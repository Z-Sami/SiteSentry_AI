from launch import LaunchDescription
from launch.actions import IncludeLaunchDescription, ExecuteProcess
from launch_ros.actions import Node
from launch.substitutions import PathJoinSubstitution
from launch_ros.substitutions import FindPackageShare

def generate_launch_description():
    
    config_dir = FindPackageShare(package='sitesentry_bringup').find('sitesentry_bringup')
    
    return LaunchDescription([
        
        # ========== MICRO-ROS AGENT (Arduino Bridge) ==========
        ExecuteProcess(
            cmd=['ros2', 'run', 'micro_ros_agent', 'micro_ros_agent', 'serial',
                 '--dev', '/dev/ttyUSB0', '-b', '115200'],
            output='screen',
            emulate_tty=True,
        ),
        
        # ========== SITESENTRY VISION NODE (YOLO + ArUco) ==========
        Node(
            package='sitesentry_vision',
            executable='vision_node',
            name='sitesentry_vision',
            output='screen',
            emulate_tty=True,
            parameters=[{
                'yolo_model': 'yolov8n.pt',
                'confidence_threshold': 0.5,
                'aruco_dict': 'DICT_6X6_250',
                'socket_tolerance': 0.10,
            }],
        ),
        
        # ========== RPLidar Node ==========
        Node(
            package='sllidar_ros2',
            executable='sllidar_node',
            name='lidar',
            output='screen',
            emulate_tty=True,

                'channel_type': 'serial',
                'serial_port': '/dev/ttyUSB1',
                'serial_baudrate': 115200,
                'frame_id': 'lidar_link',
                'angle_compensate': True,
            }],
        ),
        
        # ========== SLAM ==========

            package='slam_toolbox',
            executable='async_slam_toolbox_node',

SCENARIO 1: Robot moving forward, obstacle at 50cm

Input command: Forward 0.5 m/s
Obstacle distance: 0.50m
Safety zone: SAFE (> 0.30m)
Output: Forward 0.5 m/s ✅ (No change)

SCENARIO 2: Robot moving forward, obstacle at 25cm

Input command: Forward 0.5 m/s
Obstacle distance: 0.25m
Safety zone: WARNING (0.15-0.30m)
Reduction: (0.25 - 0.15) / (0.30 - 0.15) = 67%
Output: Forward 0.5 * 0.67 * 0.7 = 0.23 m/s ⚠️ (Slowed down)

SCENARIO 3: Robot moving forward, obstacle at 10cm

Input command: Forward 0.5 m/s
Obstacle distance: 0.10m
Safety zone: CRITICAL (< 0.15m)
Output: Forward 0.0 m/s 🛑 (STOPPED!)

SCENARIO 4: Robot at obstacle, trying to turn

Input command: Turn 1.0 rad/s (no forward)
Obstacle distance: 0.10m
Safety zone: CRITICAL

📈 PERFORMANCE PARAMETERS (Can be tuned)
================================================================================

In SITESENTRY_ros2_launch.py, adjust these:

CRITICAL_DISTANCE = 0.15 (15cm)
  - How close before STOP
  - Lower = more aggressive
  - Higher = more conservative
  
WARNING_DISTANCE = 0.30 (30cm)
  - When to start slowing down
  - Lower = less early warnings
  - Higher = more early warnings
  
SAFE_DISTANCE = 0.50 (50cm)
  - Distance for full speed
  - Lower = smaller safety zone
  - Higher = larger safety zone

SPEED REDUCTION = 0.7 (70%)
  - In code: safe_cmd.linear.x * speed_reduction * 0.7
  - Lower = slower when obstacle
  - Higher = faster when obstacle

Example tuning for tight spaces:
  CRITICAL_DISTANCE = 0.10  # 10cm
  WARNING_DISTANCE = 0.20   # 20cm
  SAFE_DISTANCE = 0.40      # 40cm

Example tuning for open spaces:
  CRITICAL_DISTANCE = 0.20  # 20cm
  WARNING_DISTANCE = 0.40   # 40cm
  SAFE_DISTANCE = 0.60      # 60cm

================================================================================
🔧 TROUBLESHOOTING
================================================================================

PROBLEM: Obstacle avoidance not working

Solution:
  1. Verify LiDAR is running:
     ros2 topic echo /scan
     (Should show data)
  
  2. Verify node is launched:
     ros2 node list | grep obstacle
     (Should show obstacle_avoidance)
  
  3. Monitor obstacle detection:
     ros2 topic echo lidar/obstacle_detected
     (Should change when move hand in front of lidar)
  
  4. Check messages going to motors:
     ros2 topic echo /cmd_vel_safe
     (Should show modified velocities)

PROBLEM: Robot stops too far from obstacle

Solution:
  Lower CRITICAL_DISTANCE and WARNING_DISTANCE
  Example: 0.15 → 0.10 (move closer before stopping)

PROBLEM: Robot too aggressive when slowing down

Solution:
  Increase SPEED_REDUCTION
  In code: change 0.7 to 0.8 or 0.9

PROBLEM: LiDAR data is noisy

Solution:
  Increase SCAN_BUFFER_SIZE in code
  From 5 to 10 (averages more scans = smoother)

PROBLEM: Robot hits obstacle anyway

Solution:
  1. Increase WARNING_DISTANCE
  2. Reduce MAX_LINEAR_SPEED
  3. Check LiDAR USB connection
  4. Verify LiDAR is spinning
