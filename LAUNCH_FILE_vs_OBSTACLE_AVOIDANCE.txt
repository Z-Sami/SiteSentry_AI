  - Starts Arduino bridge
  - Starts LiDAR scanner
  - Starts mapping (SLAM)

YOU DON'T USE ONE OR THE OTHER!

🔄 DATA FLOW - HOW THEY WORK TOGETHER
================================================================================

SCENARIO: You press "Forward" button on web dashboard

Step 1: Web Dashboard
  └─ Button pressed → Publishes to /cmd_vel
     Message: "Move forward at 0.5 m/s"

Step 2: LiDAR Obstacle Avoidance Node (YOUR NEW FILE!)
  └─ Subscribes to /cmd_vel (listens to command)
  └─ Subscribes to /scan (from LiDAR started by launch file)
  └─ Analyzes: Is there an obstacle?

Step 3: Safety Decision
  ┌─ If SAFE (obstacle far):
  │  └─ Publish /cmd_vel_safe = "Forward 0.5 m/s" (unchanged)
  │
  ├─ If WARNING (obstacle close):
  │  └─ Publish /cmd_vel_safe = "Forward 0.25 m/s" (50% slower)
  │
  └─ If CRITICAL (obstacle VERY close):
     └─ Publish /cmd_vel_safe = "Forward 0.0 m/s" (STOP!)

Step 4: Motor Controller
  └─ Reads /cmd_vel_safe (the SAFE version)
  └─ Sends to Arduino
  └─ Motors move SAFELY!

Result: Robot moves but doesn't crash! ✅

❓ COMMON QUESTIONS ANSWERED
================================================================================

Q: Do I need to edit the obstacle avoidance file?
A: NO! Use as-is. Only edit the launch file to add the Node.

Q: What if I make a mistake in the launch file?
A: Error message will tell you what's wrong.
   Fix it and relaunch.

Q: Can I test obstacle avoidance alone?
A: NO! The launch file must start first (to provide LiDAR data).

Q: What's the difference between /cmd_vel and /cmd_vel_safe?
A: /cmd_vel = original command (may hit obstacles)
   /cmd_vel_safe = modified command (safe, no obstacles)
   Motors should use /cmd_vel_safe!

Q: How do I disable obstacle avoidance temporarily?
A: Comment out the Node section in launch file:
   # Node(
   #     package='sitesentry_lidar',
   # ... )
   Then relaunch.

Q: Will it slow down my system?
A: NO! Obstacle avoidance runs independently in parallel.

Q: Can I use just the launch file without obstacle avoidance?
A: YES! Just don't add that Node section.
   Robot will work but without safety.

Q: Which file actually controls the robot?
A: Launch file controls everything:
   - It STARTS the obstacle avoidance node
   - Obstacle avoidance MODIFIES motor commands
   - Motor controller EXECUTES the safe commands
